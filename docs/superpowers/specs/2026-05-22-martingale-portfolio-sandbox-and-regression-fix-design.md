# Martingale Portfolio Sandbox and Regression Fix Design

**Date:** 2026-05-22
**Status:** Draft for user confirmation
**Owner:** grid_binance martingale backtest/live portfolio system

## 背景

当前马丁回测系统已经具备自动搜索单策略候选、生成系统推荐组合、保存组合资金曲线/回撤曲线的基础能力，但本轮验证暴露出多处回归和缺口：

1. 推荐币种入口错误固定为 18 个币种，而用户要求从“自 2023-01-01 起有本地 1m 数据且交易量排名前 50”的币种中筛选，全部填入前端供用户删减。
2. 7 币种任务只落库 DOGEUSDT 候选，18 币种任务只落库 LINKUSDT/NEARUSDT 候选；不是前端隐藏，而是 Worker 精测/保存阶段筛选过窄。
3. 18 币种任务配置为 `usd_m_futures + long_and_short`，但候选中混入 `spot`，说明候选生成或兜底路径没有强制继承任务市场。
4. 历史 `validation-7-symbol-v2` / `validation-18-symbol-v2` 可跑出年化 50%+ 且回撤低于 20% 的候选；当前 v4 改动后收益明显下降，说明收益优先搜索和候选保留逻辑发生回归。
5. 图表仍是窄版 sparkline，无法鼠标悬停查看日期和数值。
6. 系统自动组合 TopN 只是推荐，用户需要在本轮直接实现“人工组合沙盒”：选择某个自动组合后，可手动加入/移除已回测策略、调整权重/杠杆、重新计算整个组合回测结果，再决定是否发布实盘。
7. 实盘发布链路需要验证：最终确认的自定义组合必须携带候选完整参数、权重、杠杆、市场、方向到 live portfolio / item 记录，并且确认启动后参数仍可供交易执行层消费。

## 目标

本轮必须完成一个可用闭环：

- 用户选择币种池、方向、风险档位后，系统能搜索并展示尽可能多的有效单策略候选。
- 在最大回撤约束内，优先恢复并寻找年化收益更高、收益/回撤比更优的候选。
- 系统自动组合不再硬性追求固定成员数，而是在收益尽量高、回撤尽量低、成员数越多越好的软约束下给出推荐组合。
- 用户可从推荐组合进入“人工组合沙盒”，手动加入已回测候选，调整权重/杠杆，快速重算组合收益曲线、回撤曲线、年化收益、最大回撤和交易明细摘要。
- 用户确认后，可把沙盒最终组合保存并发布到实盘待确认组合。

## 非目标

- 本轮不实现直接修改单策略内部马丁参数后的即时重算。若用户修改间隔、倍率、层数、止盈止损等单策略内部参数，必须新建候选回测任务。
- 本轮不引入外部行情实时下载；仍使用本地只读 SQLite 行情库。
- 本轮不承诺一定找到年化 50%+ 组合，因为结果由行情和风险约束决定；但搜索和筛选逻辑必须不再漏掉历史可找到的高收益候选。

## 需求详情

### 1. 推荐币种池

前端自动搜索向导的推荐按钮必须改为：

- 查询本地行情库中 `market_universe.market_type='futures_usdt_perp'`、`quote_asset='USDT'` 的币种。
- 必须有 `klines` 的 `1m` 数据覆盖 `2023-01-01` 附近。
- 必须近期仍有数据，避免已下架或数据过期币种。
- 按 `market_universe.volume_24h` 降序取前 50。
- 前端将这些币种全部填入白名单 textarea，用户自行删减。
- 保留手动输入能力。
- 如果本地行情库查询失败，才使用内置兜底币种，并提示用户这是兜底结果。

### 2. 单策略候选搜索与保存

Worker 必须修复候选过少问题：

- 对每个输入币种，最终候选表应尽量保存 Top10。
- 对于没有满足严格回撤约束的币种，应保存“最佳失败/接近候选”或提供清晰淘汰诊断，而不是静默消失。
- 正收益候选优先；负收益候选默认不进入最终可发布候选，但可进入诊断统计。
- `long_and_short` 必须生成真正双腿策略：long 与 short 可使用不同间隔、倍率、层数、止盈止损和权重。
- 任务市场为 `usd_m_futures` 时，所有候选策略腿必须是 `usd_m_futures`，不得混入 `spot`。
- 杠杆必须完整参与回测，收益和回撤按杠杆前实际本金/保证金口径计算。
- 候选 summary 必须包含并前端展示：`max_leverage_used`、各腿 leverage、market、direction、long/short legs、年化收益、最大回撤、收益/回撤比、交易数、评分。

### 3. 恢复收益优先搜索基线

本轮修复后至少要用历史高收益任务作为回归基准：

- `validation-7-symbol-v2` 曾有 BTCUSDT long_short 年化约 68.97%、最大回撤约 19.98%。
- `validation-18-symbol-v2` 曾有 XRPUSDT long_short 年化约 60.26%、最大回撤约 15.21%。
- 新逻辑不能因为组合成员数、过度保守筛选、balanced 默认权重或保存阶段过滤而漏掉同类候选。
- 风险档位仍控制最大回撤：保守 20%、均衡 25%、激进 30%；如果用户手动设置最大回撤，则按用户值作为硬约束。
- 排名目标：在硬回撤约束下优先收益/回撤比和增长稳定性，其次考虑成员数与低相关性。

### 4. 自动组合器

自动组合不应硬性限制为 10 币种或 10 成员。组合器应：

- 从已保存的有效候选池中组合。
- 允许同一币种出现多个不同参数策略，但单币种总权重默认不超过 80%。
- 以最大回撤约束为硬门槛。
- 优先最大化年化收益、收益/回撤比、曲线稳定性。
- 成员数越多越好作为软奖励，用于降低单一策略风险，但不能为了凑成员数牺牲明显收益。
- 输出 Top10 自动组合，每个组合包含：成员、权重、杠杆、收益曲线、回撤曲线、交易明细预览、年化、最大回撤、收益/回撤比、成员贡献。

### 5. 人工组合沙盒

新增“组合沙盒 / Portfolio Sandbox”：

- 用户可从任一自动组合点击“编辑组合”。
- 沙盒初始化为该组合的成员、权重、杠杆、enabled 状态。
- 用户可从当前任务已回测候选列表中加入任意候选。
- 用户可移除成员、启用/禁用成员、调整权重、调整杠杆。
- 权重合计必须为 100% 才能重算或发布。
- 点击“重新计算组合表现”后，后端基于候选已保存曲线快速合成组合结果。
- 重算结果必须返回：组合资金曲线、组合回撤曲线、年化收益、最大回撤、收益/回撤比、成员贡献、交易明细预览、集中度警告、是否满足最大回撤约束。
- 若用户只调整权重/杠杆/成员启用状态，可快速重算；若要改单策略内部参数，提示必须重新跑单策略回测。
- 用户可保存沙盒结果为“自定义组合候选”。
- 用户可直接将沙盒最终组合发布到实盘待确认组合。

### 6. 图表交互与布局

资金曲线和回撤曲线必须升级：

- 图表区域改为宽版，占据结果区主要宽度，不再挤在左侧小块。
- 鼠标悬停显示 tooltip：日期、资金值、收益百分比、回撤百分比。
- 图表支持候选和组合两种 summary。
- 组合沙盒重算后，也使用同一图表组件展示。
- 交易明细至少展示前 100 条预览，并保留总条数提示。

### 7. 实盘发布链路

发布必须使用最终确认的组合，而不是旧自动推荐组合：

- 单候选发布继续可用。
- 批量发布使用当前沙盒/篮子最终状态。
- 每个 item 必须写入：candidate_id、symbol、weight_pct、leverage、enabled、parameter_snapshot、metrics_snapshot。
- `parameter_snapshot` 必须是完整候选 `config`，包含 long/short 双腿参数、market、margin_mode、leverage、spacing、sizing、take_profit、stop_loss、entry_triggers。
- 发布后 portfolio 状态为 `pending_confirmation`。
- 确认启动后 portfolio 状态变为 `running`，参数快照不能丢失。
- 如果当前交易执行层还没有消费 live portfolio 参数，必须明确在代码/测试中暴露断点或 TODO 不允许；本轮要验证到服务边界。

## 验收标准

### 自动搜索验收

- 推荐按钮返回最多前 50 高流动性币种，并全部填入白名单。
- 新跑 7 币种 long+short futures 回测后，结果不应只出现 DOGEUSDT；若某币种无可用候选，页面必须显示淘汰诊断。
- 新跑 18/50币种池回测后，候选表应按币种分组展示 Top10 或清晰失败原因。
- 不得出现 `spot` 候选。
- 候选表展示杠杆。

### 组合验收

- 自动组合 Top10 可查看曲线和明细。
- 组合成员数量由目标函数决定，不硬性固定 10。
- 至少验证一组高收益候选可进入组合池。
- 组合结果曲线有 500 个左右采样点，回撤曲线有对应采样点。

### 沙盒验收

- 可从自动组合进入编辑。
- 可手动加入一个已回测候选。
- 调整权重到 100% 后可重新计算。
- 重算后曲线、年化、回撤发生合理变化。
- 可保存/发布沙盒最终组合。

### 图表验收

- 鼠标 hover 显示日期和数值。
- 图表宽度明显扩大。
- 候选、自动组合、自定义沙盒组合都能展示。

### 发布验收

- 发布后的 `martingale_portfolios` 与 `martingale_portfolio_items` 有记录。
- item 的 `parameter_snapshot` 包含完整 long/short 双腿配置。
- `confirm-start` 后状态为 `running`，参数仍保留。

## 风险与约束

- 大币种池深搜耗时会增加，需要保留进度事件和可见状态。
- 组合层重算可快速完成，但如果候选缺少曲线，则必须拒绝重算并提示原因。
- 杠杆调整如果只做组合层缩放，必须明确这是组合层资金缩放；若要改变单策略内部成交路径，必须重新回测。
- 当前本地行情库时间戳可能超过真实当前日期，所有时间窗口仍以任务配置和数据可用范围为准。
