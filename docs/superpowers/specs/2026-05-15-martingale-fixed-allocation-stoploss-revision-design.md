# 马丁固定多空配比与止损重算修订设计

**状态:** 待确认  
**日期:** 2026-05-15  
**目标分支:** `feature/full-v1`  
**背景:** 当前动态多空回测版本出现收益普遍为负、结果劣于前一稳定版本的问题。用户要求先关闭动态多空权重模型，恢复可解释、可手动调整的固定组合级多空配比；同时重新审视止损、手续费、滑点、强平、止损频率对收益的影响，以“优先满足最大回撤限制，在此基础上追求最高收益”为核心搜索目标。

## 1. 本轮目标

1. 暂停动态多空权重模型，不再按行情状态频繁切换 long/short 权重。
2. `long + short` 模式改为组合级固定多空配比：
   - 保守：Long `80%` / Short `20%`
   - 均衡：Long `60%` / Short `40%`
   - 激进：Long `50%` / Short `50%`
   - 用户可手动调整，Long + Short 必须等于 `100%`。
3. 最大回撤限制采用“优先硬约束”：
   - 优先输出最大回撤小于等于用户设置值的候选。
   - 若没有合规候选，展示最接近候选，但标记“不建议实盘”。
   - 不合规候选不能默认进入一键实盘发布。
4. 止损从“过早割肉”改为“层数止损 + ATR 波动止损 + 组合总回撤兜底”。
5. 明确区分移动止盈回撤与止损：移动回撤只在整体止盈激活后生效，不能导致未加仓前提前止损。
6. 搜索目标从“动态配比模型得分最高”调整为“在回撤限制下，收益、止损频率、手续费磨损、样本稳定性综合最优”。
7. 前端结果必须能解释亏损来源，包括止损次数、止损损耗、手续费、滑点、强平损耗、交易次数和年化收益。

## 2. 非目标

1. 本轮不继续优化动态多空行情识别模型。
2. 本轮不引入资金费率作为评分或调仓依据。
3. 本轮不承诺一定找到年化 `50%+` 且回撤 `<20%` 的组合；系统以该目标作为搜索方向，并真实展示是否达成。
4. 本轮不改造交易所实盘执行引擎的仓位模式；只保证回测和发布配置中的参数语义清晰。

## 3. 固定多空配比规则

### 3.1 组合级而非币种级

固定配比在组合层生效：

- 总 long 预算 = 组合预算 × long 权重。
- 总 short 预算 = 组合预算 × short 权重。
- 系统在 long 预算池内选择 long 候选，在 short 预算池内选择 short 候选。
- 不要求每个币种都同时拥有 long 和 short。
- 如果某个方向没有足够优质候选，允许该方向出现未分配预算，并在结果中提示“方向预算未完全利用”。

### 3.2 默认值

| 风险档位 | Long | Short | 最大回撤默认值 |
| --- | ---: | ---: | ---: |
| 保守 | 80% | 20% | 20% |
| 均衡 | 60% | 40% | 25% |
| 激进 | 50% | 50% | 30% |

用户切换风险档位时：

- 若用户没有手动改过多空比例，自动切换默认比例。
- 若用户已手动改过多空比例，不覆盖用户输入，除非用户点击“恢复风险档位默认值”。
- 若用户没有手动改过最大回撤限制，自动切换默认最大回撤。
- 若用户已手动改过最大回撤限制，不覆盖用户输入。

### 3.3 手动比例校验

- Long + Short 必须等于 `100%`。
- 任一方向不能为负数。
- `long + short` 模式下允许 `100/0` 或 `0/100`，但 UI 需要提示这等价于单方向组合。
- `long only` 固定为 `100/0`。
- `short only` 固定为 `0/100`。

## 4. 止损模型修订

### 4.1 单策略止损

单策略止损由两个条件共同定义，谁更宽松不代表立即触发；系统应选取合理止损边界，避免频繁止损：

1. **层数止损**
   - Long：价格跌破最后一层补仓价后，再额外偏离 `extra_stop_spacing_multiplier × spacing_pct` 才允许止损。
   - Short：价格涨破最后一层补仓价后，再额外偏离 `extra_stop_spacing_multiplier × spacing_pct` 才允许止损。
   - 默认搜索范围：`[1.0, 1.5, 2.0, 2.5, 3.0]`。
2. **ATR 止损**
   - 根据近期 ATR 放宽或收紧止损距离。
   - 默认搜索范围：`atr_stop_multiplier = [2.0, 2.5, 3.0, 3.5, 4.0]`。
   - ATR 止损不能小于最后一层外的最小保护距离，避免还没完成马丁层级就止损。

最终单策略止损价取更能避免噪声止损的一侧，但仍必须受组合最大回撤兜底约束。

### 4.2 组合总回撤兜底

- 组合资金曲线最大回撤达到用户设置值时，该组合候选判定为不通过。
- 回测中可以继续计算完整结果，但结果必须标记 `max_drawdown_limit_passed=false`。
- 实盘发布默认禁止发布不通过候选，除非未来增加显式风险确认流程。

### 4.3 止损频率惩罚

候选评分必须惩罚频繁止损：

- 止损次数越多，评分越低。
- 止损损耗占总投入比例越高，评分越低。
- 如果收益主要来自少数大盈利、但大量小止损磨损，标记“止损磨损偏高”。
- 如果交易次数过少导致收益不稳定，标记“样本不足”。

## 5. 杠杆与资金口径

1. 收益率与回撤按马丁策略总保证金预算计算。
2. 例如首单保证金 `10U`、倍投 `2`、4 层，总保证金预算为 `10 + 20 + 40 + 80 = 150U`。
3. 杠杆只放大名义仓位，不改变保证金预算口径。
4. 手续费与滑点按名义成交额计算。
5. 强平风险、止损风险必须考虑杠杆后的名义价格波动影响。
6. 回测结果必须展示杠杆、总保证金预算、名义最大仓位、手续费、滑点和强平风险。

## 6. 参数搜索目标

### 6.1 搜索约束

系统应按以下顺序搜索和筛选：

1. 过滤爆仓或强平风险不可接受的候选。
2. 优先保留最大回撤小于等于用户设置值的候选。
3. 在合规候选中追求更高年化收益。
4. 同等收益下选择更低回撤、更低止损频率、更低手续费磨损、更稳定样本。
5. 如果没有合规候选，展示最接近限制的 fallback，但标记“不建议实盘”。

### 6.2 收益目标展示

系统展示目标达成情况：

- 年化收益目标：默认 `50%`。
- 最大回撤目标：用户设置值，保守默认 `20%`。
- 输出字段：
  - `annualized_return_pct`
  - `max_drawdown_pct`
  - `target_annualized_return_passed`
  - `max_drawdown_limit_passed`
  - `can_recommend_live`
  - `failure_reason` 或 `warning_reason`

若年化收益未达到 `50%`，但回撤合规，仍可展示为“低收益合规候选”，不能夸大为达成目标。

## 7. 100 分评分规则

score 固定为 `0 ~ 100`，不能出现极大数字。

建议权重：

- 年化收益：`35` 分
- 最大回撤：`30` 分
- 止损频率与强平风险：`15` 分
- 手续费、滑点、换仓磨损：`10` 分
- 交易活跃度与样本稳定性：`10` 分

评分要求：

- 超过最大回撤限制的候选，分数必须显著下降。
- 频繁止损候选，分数必须显著下降。
- 年化收益为负的候选，不能获得高分。
- 分数展示保留 `1` 位小数。

## 8. 前端交互要求

### 8.1 向导参数

前端向导默认只要求用户填写：

- 币种白名单/黑名单。
- 市场：spot 或 futures。
- 方向：long only、short only、long + short。
- 风险档位：保守、均衡、激进。
- 最大回撤限制，按风险档位自动填入，可手动修改。
- 多空比例，仅在 `long + short` 模式显示，可手动修改。

高级参数可折叠展示：

- 加仓间距搜索范围。
- 倍投倍率搜索范围。
- 层数搜索范围。
- 止盈搜索范围。
- 杠杆搜索范围。
- 层数止损倍数。
- ATR 止损倍数。

### 8.2 结果解释

结果页必须清晰展示：

- 每币种 Top10。
- 组合 Top10。
- 年化收益率。
- 最大回撤。
- score 100 分制。
- long/short 固定目标比例。
- 实际资金分配比例。
- 止损次数。
- 止损损耗。
- 手续费。
- 滑点。
- 强平/强制退出次数。
- 交易次数。
- 资金曲线。
- 回撤曲线。
- 交易明细入口。
- 是否达到 `年化 50%` 与 `回撤限制`。
- 是否建议实盘。

### 8.3 风险提示

当结果收益为负或不达标时，UI 必须直接说明：

- “该候选收益为负，不建议实盘”。
- “该候选超过最大回撤限制，不建议实盘”。
- “该候选止损频率过高，可能被手续费和滑点磨损”。
- “未找到同时满足年化目标和回撤限制的组合”。

## 9. API 与数据契约

### 9.1 请求新增/调整字段

回测创建请求应包含：

```json
{
  "risk_profile": "conservative",
  "direction": "long_and_short",
  "max_drawdown_limit_pct": 20,
  "target_annualized_return_pct": 50,
  "allocation_mode": "fixed_by_risk_profile",
  "long_weight_pct": 80,
  "short_weight_pct": 20,
  "dynamic_allocation_enabled": false,
  "stop_model": {
    "kind": "layer_plus_atr",
    "extra_stop_spacing_multipliers": [1.0, 1.5, 2.0, 2.5, 3.0],
```

## 10. 验证标准

必须新增或更新验证，证明：

1. 动态多空默认关闭，请求中 `dynamic_allocation_enabled=false`。
2. 保守/均衡/激进默认多空比例分别为 `80/20`、`60/40`、`50/50`。
3. 手动多空比例总和必须为 `100%`，否则前端或 API 拒绝。
4. 最大回撤采用优先硬约束：无合规候选时返回 fallback，并标记不建议实盘。
5. 移动止盈回撤不会在未达到止盈前触发止损。
6. 单策略止损必须在最后一层之外结合 ATR 计算，不能早于完成马丁层级。
7. score 始终在 `0 ~ 100`。
8. 年化收益为负的候选不能显示为推荐实盘。
9. 结果中能看到止损次数、手续费、滑点、年化收益、最大回撤和实盘推荐状态。
10. 运行一次 BTCUSDT + ETHUSDT、long + short、保守、最大回撤 `20%` 的回测，能看到 Top10 或 fallback Top10，并能解释为什么达标或不达标。

## 11. 与旧动态多空设计的关系

`2026-05-14-martingale-dynamic-long-short-allocation-design.md` 中的动态权重模型暂时冻结，不作为默认回测逻辑。保留相关设计文档，但本轮实现以本文档为准：

- 默认关闭动态权重。
- 不展示动态权重曲线作为核心结果。
- 可保留历史字段，但必须标记为 disabled 或 experimental。
- 后续若重新启用动态多空，必须重新走 spec/plan 确认流程，并先证明固定配比版本稳定盈利或至少结果可解释。
