# 马丁收益优先自动搜索与组合回测重构设计

**状态:** 待确认  
**日期:** 2026-05-15  
**目标分支:** `feature/full-v1`  
**背景:** 当前马丁回测经历多轮动态多空、止损和组合逻辑调整后，出现大量负收益结果。用户要求重构为“少输入、自动搜索、正收益优先、可用于实盘”的马丁参数发现系统。

## 1. 用户目标

用户只输入少量高层条件：

- 交易对列表，例如 `BTCUSDT, ETHUSDT, SOLUSDT`。
- 市场固定为 USDT-M 合约，移除现货选项。
- 方向：只做多、只做空、多空双向。
- 风险档位：保守、均衡、激进。

系统自动搜索并回测所有核心参数，输出：

- 每个交易对的单策略 Top 10。
- 从所有单策略候选中二次组合出的组合策略 Top 3。
- 每个结果必须是扣除手续费、滑点、保证金、维持保证金和强平风险后的保守真实结果。
- 负收益结果不得进入 Top 10 或 Top 3。

最终目标是找到在可控回撤内收益/回撤比最高、增长稳定、具备实盘参考价值的马丁策略组合。

## 2. 默认用户输入

### 2.1 必填输入

默认向导只保留以下输入：

- `symbols`: 交易对列表。
- `direction`: `long`、`short`、`long_short`。
- `risk_profile`: `conservative`、`balanced`、`aggressive`。

### 2.2 市场固定

- 市场默认且固定为 USDT-M futures。
- UI 不再展示 spot/现货选择。
- 后端 contract 若仍兼容 `market` 字段，应默认写入 `futures` 并拒绝现货马丁自动搜索任务。

### 2.3 风险档位

风险档位自动映射最大回撤限制：

- 保守：最大回撤 20%。若找不到正收益候选，可放宽到 25%。
- 均衡：最大回撤 25%。若找不到正收益候选，可放宽到 30%。
- 激进：最大回撤 30%。不再放宽。

放宽规则只放宽回撤限制，不允许负收益候选通过。

## 3. 自动搜索参数

以下参数不再要求用户手动填写，而是由系统自动组合搜索：

- 杠杆 `leverage`。
- 加仓间隔 `spacing_bps`。
- 加仓倍率 `order_multiplier`。
- 最大层数 `max_legs`。
- 整体止盈范围 `take_profit_bps`。
- 尾部保护/灾难退出线 `tail_stop_bps` 或等价风险退出参数。
- 多空双向时的 long/short 仓位比例 `long_weight_pct` / `short_weight_pct`。

多空双向时，多头和空头必须允许使用不同参数，例如：

- 多头间隔 120 bps，空头间隔 180 bps。
- 多头最大层数 5，空头最大层数 4。
- 多头止盈 90 bps，空头止盈 120 bps。
- 多头权重 70%，空头权重 30%。

## 4. 分阶段搜索流程

### 4.1 粗筛阶段

粗筛用于覆盖完整参数边界，不能只随机抽少量参数。

要求：

- 每个交易对独立搜索，不能所有交易对共用同一组参数。
- 覆盖低/中/高间隔、低/中/高倍率、不同最大层数、2–10 倍杠杆、不同止盈范围、多空比例关键档位。
- 保留多目标候选池，不只保留收益最高者。
- 候选池至少保留以下类型：
  - 收益/回撤比高。
  - 收益为正且回撤接近限制。
  - 收益较低但曲线稳定。
  - 杠杆较低且强平距离安全。
- 粗筛失败时，应进入风险放宽流程，而不是直接返回负收益结果。

### 4.2 精筛阶段

精筛围绕粗筛优胜区间扩展，不只重复粗筛点位。

示例：

- 粗筛发现 `spacing_bps = 120` 表现好，精筛应测试 `90, 100, 110, 120, 130, 150` 等邻近值。
- 粗筛发现 `max_legs = 5` 表现好，精筛应测试 `4, 5, 6`。
- 粗筛发现多空比例 `70/30` 表现好，精筛应测试 `60/40, 65/35, 70/30, 75/25, 80/20`。

### 4.3 最终验证阶段

Top 10 与 Top 3 只能来自完整最终验证结果。

最终验证必须满足：

- 使用完整 1m K线。
- 回测区间为 `2023-01-01` 到当前日期所在月份的上个月月底。
- 例如当前日期为 `2026-05-15`，终点为 `2026-04-30`。
- 不允许只用前几天或局部窗口结果生成最终排名。
- 若交易对 1m 数据不足，应标记“数据不足”，不得伪造候选。

## 5. 时间切分与防过拟合

完整区间自动切分为：

- train：约 70%。
- validate：约 15%。
- test：约 15%。
- stress windows：自动识别或固定抽取趋势上涨、趋势下跌、高波动、快速回撤窗口。

排序时不能只看全周期总收益，还必须检查：

- train/validate/test 都不能长期亏损。
- validate/test 中表现明显崩坏的候选降权或淘汰。
- 单一短窗口贡献绝大部分收益的候选降权。
- 月度收益过度集中、资金曲线长期横盘后突然暴涨的候选降权。

## 6. 保守真实撮合与成本模型

回测必须按合约逐仓思路模拟。

必须计入：

- 开仓手续费。
- 平仓手续费。
- 滑点。
- 杠杆后的名义仓位。
- 实际保证金占用。
- 维持保证金估算。
- 强平距离和强平风险。

杠杆收益与回撤计算规则：

- 例如首层保证金 10U、2 倍杠杆，则名义仓位为 20U。
- 盈亏按名义仓位价格变化计算。
- 收益率、最大回撤、资金曲线按该策略计划投入的总保证金计算。
- 多层马丁的计划投入资金为各层保证金之和，例如 `10 + 20 + 40 + 80 = 150U`。
- 不允许只按首单 10U 计算收益率或回撤。

## 7. 止损与尾部保护

本轮不采用过近移动止损作为主要风险控制。

默认采用“尾部保护优先”：

- 主要风险控制来自最大层数、杠杆、保证金占用、强平距离、组合最大回撤。
- 每个候选必须有灾难退出线，避免接近强平才处理。
- 灾难退出线由系统搜索，但原则是少触发、不频繁磨损。
- 如果候选依赖频繁灾难止损维持回撤，应降权或淘汰。
- 不能出现“加仓间隔 1%，但 0.4% 回撤就止损”的错误逻辑。

## 8. 单交易对 Top 10

每个交易对输出最多 Top 10。

进入 Top 10 的硬性条件：

- 成本后收益为正。
- 未触发强平。
- 未超过当前风险档位或允许放宽后的最大回撤。
- 最终验证使用完整 1m K线。
- 交易数量、资金曲线和交易明细完整可查看。

每个候选至少展示：

- 交易对、方向、风险档位、是否使用放宽回撤。
- 杠杆、间隔、倍率、最大层数、止盈、尾部保护参数。
- 多空双向时分别展示 long 参数、short 参数、long/short 权重。
- 年化收益、总收益、最大回撤、收益/回撤比。
- 月度胜率、最大连续亏损、交易次数、手续费、滑点成本。
- 强平距离摘要、灾难退出次数、评分。
- 资金曲线、回撤曲线、交易明细入口。

## 9. 100 分制评分

最终评分必须为 0–100 分，不能输出极大 score。

评分原则：

- 负收益：淘汰，不评分。
- 超回撤：淘汰，不评分。
- 强平或接近强平：淘汰或重罚。
- 收益/回撤比：核心加分项。
- 年化收益：加分项，但不能压过风险约束。
- 月度稳定性：加分项。
- 资金曲线平滑度：加分项。
- 高杠杆、高换手、高成本、频繁灾难退出：扣分项。
- 单一窗口收益过度集中：扣分项。

排序必须体现“实盘安全优先”：收益很高但风险明显失控的策略不得排在前列。

## 10. 多空双向逻辑

当方向为 `long_short` 时，系统必须真正同时模拟多头与空头，而不是只返回 long。

要求：

- long 和 short 独立开仓、加仓、止盈、尾部保护。
- long 和 short 可以拥有不同参数。
- long 和 short 共享该策略实例的计划资金，但按搜索出的比例分配。
- 多空比例由搜索决定，不固定为单一默认值。
- 若某交易对 short 长期风险过高，搜索可以将 short 权重降到很低，但结果必须明确显示原因。
- 多空双向候选的收益、回撤、成本、交易明细必须合并展示，同时允许查看 long/short 分项贡献。

## 11. 二次组合 Top 3

假设用户输入 7 个交易对，每个交易对输出 Top 10，则组合优化从最多 70 个候选中搜索组合策略。

组合搜索要求：

- 允许同一交易对多个策略同时入选。
- 自动搜索每个策略的资金权重。
- 不强制所有交易对都入选。
- 表现差的交易对可以被舍弃。
- 表现好的交易对可以入选多个策略。

默认组合约束：

- 单策略权重默认不超过 20%。
- 单交易对总权重默认不超过 35%。
- 权重合计必须为 100%。
- 组合必须满足风险档位对应的最大回撤限制；若单策略阶段使用放宽限制，组合结果也必须明确标记。

组合 Top 3 排名依据：

- 组合收益/回撤比。
- 组合年化收益。
- 组合资金曲线稳定性。
- 策略间相关性和回撤重叠程度。
- 集中度风险。
- 交易成本与换手。

组合结果至少展示：

- 入选策略列表。
- 每个策略权重。
- 每个交易对总权重。
- 组合年化收益、总收益、最大回撤、收益/回撤比、评分。
- 组合资金曲线、组合回撤曲线。
- 单策略贡献拆解。
- 组合交易明细入口。
- 可用于后续实盘发布的配置摘要。

## 12. UI 交互要求

向导页面必须让普通用户看得懂。

### 12.1 创建回测

用户只需要完成：

1. 输入交易对。
2. 选择方向。
3. 选择风险档位。
4. 点击“开始自动搜索”。

页面应明确提示：

- 系统会自动搜索杠杆、间隔、倍率、层数、止盈、尾部保护、多空比例。
- 回测区间自动为 `2023-01-01` 到上个月月底。
- 结果只展示成本后正收益候选。
- 若某交易对没有合格策略，会显示失败原因。

### 12.2 结果管理

必须支持：

- 查看任务进度。
- 查看每个交易对 Top 10。
- 查看组合 Top 3。
- 删除回测任务或结果。
- 查看资金曲线、回撤曲线。
- 鼠标悬停图表时显示日期、收益百分比、回撤百分比。
- 查看交易明细。
- 查看候选为什么入选或被淘汰的摘要。

## 13. API 与数据 contract

自动搜索任务 payload 至少包含：

- `symbols`
- `market = futures`
- `direction`
- `risk_profile`
- `per_symbol_top_n = 10`
- `portfolio_top_n = 3`
- `time_range_mode = auto_since_2023_to_last_month_end`
- `search_mode = staged`
- `execution_model = conservative_futures_isolated`

结果 contract 至少支持：

- 按 symbol 分组的 Top 10。
- 组合 Top 3。
- 每个候选的参数、指标、评分、成本、强平风险、曲线、交易明细引用。
- 每个淘汰交易对或淘汰候选的失败原因。

## 14. 实盘衔接边界

本 spec 只要求回测与组合配置能生成可用于实盘的参数摘要。

发布到实盘前必须仍有确认步骤：

- 展示策略参数。
- 展示资金权重。
- 展示杠杆。
- 展示风险摘要。
- 用户确认后再调用实盘发布 API。

不得因为回测完成就自动实盘下单。

## 15. 验证要求

实施完成后必须验证：

- UI 默认只需要交易对、方向、风险档位。
- 市场固定为 futures，不再要求用户选择现货。
- 当前日期 `2026-05-15` 时，默认终点为 `2026-04-30`。
- 自动搜索 payload 包含 Top 10、Top 3、staged search、conservative futures isolated 模型。
- 单交易对结果只展示正收益候选。
- 找不到正收益时按风险档位最多放宽到下一档。
- 多空双向确实同时模拟 long 和 short，且参数可不同。
- 杠杆收益/回撤按计划总保证金计算。
- 最终验证不只使用局部日期。
- 图表 tooltip 显示日期、收益、回撤。
- 交易明细可查看。
- 结果可删除。
- 组合 Top 3 可从所有单策略候选中自动搜索权重。
- 后端 Rust 测试、worker 测试、前端 contract 测试、前端 build 通过。

## 16. 非目标

- 本轮不承诺找到一定年化 50% 且回撤 20% 以内的真实策略；系统只能真实搜索并报告结果，不能伪造收益。
- 本轮不接入资金费率作为排序依据。
- 本轮不做 GPU 加速承诺。
- 本轮不自动实盘下单。
- 本轮不恢复动态行情实时调仓；多空比例先通过回测搜索得到，用于候选配置。
