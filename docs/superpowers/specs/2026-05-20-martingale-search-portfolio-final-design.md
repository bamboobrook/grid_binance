# 马丁回测搜索与组合最终修复设计

**状态:** 已确认执行  
**日期:** 2026-05-20  
**执行者:** Codex 直接实施

## 目标

修复当前马丁回测系统在多币种、多空双向、候选保留、组合策略、收益/回撤排序上的核心偏差，使系统能通过真实回测筛选：

1. 每个币种的单策略 Top10。
2. 多策略组合 Top3。
3. 组合中单币种资金权重不超过 80%。
4. `long_short` 必须是真双向：long 与 short 均参与回测、参数搜索、止损搜索和收益/回撤计算。
5. 负收益候选不能进入合格候选与组合。
6. 回测结果必须保留年化、资金曲线、回撤曲线、交易明细、杠杆与计划保证金。

## 用户输入

前端用户只需要输入/选择：

- 交易对列表。
- 市场默认合约。
- 方向：`long`、`short`、`long_short`。
- 风险档位：保守、均衡、激进。
- 可选最大回撤限制。

其余参数由系统搜索，包括：

- 杠杆。
- 加仓间隔。
- 加仓倍率。
- 最大层数。
- 止盈范围。
- 止损范围。
- `long_short` 的多空权重。
- `long_short` 的 long/short 独立参数组合。

## 排序原则

单策略候选排序以以下原则为准：

1. 收益为正。
2. 最大回撤满足风险档位/用户限制。
3. 年化收益尽量高。
4. 收益/回撤比尽量高。
5. 曲线增长更稳定。
6. 交易次数不过度异常，避免手续费磨损导致虚假高频。

组合策略排序以以下原则为准：

1. 至少两个成员。
2. 组合总资金权重为 100%。
3. 单币种总权重不超过 80%。
4. 优先跨币种分散组合。
5. 组合最大回撤满足限制。
6. 组合年化收益与收益/回撤比尽量高。
7. 组合资金曲线由成员资金曲线按权重合成，不允许只挑一个单策略冒充组合。

## long_short 要求

当方向为 `long_short`：

- 每个候选必须包含 long leg 与 short leg。
- long/short 可以使用不同间隔、止盈、止损、最大层数、倍率、杠杆、权重。
- 止损参数必须参与搜索，不得固定一个导致全部负收益的数值。
- 如果某个币种在初始参数空间下没有正收益候选，系统应扩展/保留诊断，不得静默只返回 BTC。

## 验收标准

以 BTCUSDT + ETHUSDT、`long_short`、balanced 为最小冒烟：

- 任务完成状态为成功。
- 候选列表同时包含 BTCUSDT 和 ETHUSDT，且每个币种尽量返回 Top10。
- 候选方向都是 `long_short`，且含 long/short 双腿。
- 候选有 `annualized_return_pct`、`planned_margin_quote > 0`、资金曲线、回撤曲线、交易明细。
- 组合 Top3 至少 Top1 是跨 BTCUSDT+ETHUSDT 的真实资金组合。
- Top1 组合 `portfolio_unique_symbol_count >= 2`。
- 任一单币种组合权重不得超过 80%。
- 组合资金曲线第一点为 10000 附近，最后一点与收益率匹配。
- 负收益候选不得进入组合。

## 非目标

- 不保证任何参数空间必然得到年化 50%+，但系统必须尽力搜索并正确排序。
- 不新增交易所实盘执行逻辑。
- 不修改无关 3000 端口服务。
