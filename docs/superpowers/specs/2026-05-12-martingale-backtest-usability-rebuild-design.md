# 马丁回测系统可用性重构规格

**状态:** 已确认，包含组合批量实盘发布 API 与清理全部回测任务  
**日期:** 2026-05-12  
**目标分支:** `feature/full-v1`  
**触发原因:** 当前网页端马丁回测仍不可用：时间范围错误、参数仍需大量手填、回测结果不可见、候选与图表缺失、用户无法理解下一步。

## 1. 目标

把当前“缝补式”的马丁回测页面重构为一个人类可用的自动化流程：

1. 用户默认只填写：币种、预设参数/风险档位、市场、方向。
2. 系统自动计算回测时间：`2023-01-01` 到当前日期所在月份的上个月最后一天。
3. 系统自动为每个币种搜索 Top 5 参数组合。
4. 回测完成后，页面必须清晰展示每个币种完整 Top 5、图表、风险摘要、候选参数、推荐权重与推荐杠杆。
5. 用户可以从每个币种 Top 5 中选择一个或多个策略实例，组合成资金权重合计 100% 的马丁组合。
6. 页面必须明确告诉用户“下一步怎么做”：筛选、组合、调整权重/杠杆、批量发布到实盘组合。
7. 本轮必须完成组合批量实盘发布 API：把组合篮子一次性保存为实盘组合，并生成对应的多个策略实例或发布意图，不允许只停留在单候选发布。
8. 按用户确认，清理当前所有回测任务和结果，让页面从干净状态重新开始。

## 2. 当前必须修复的问题

### 2.1 自动时间错误

当前前端显示结束日期为 `2025-04-30`，但当前日期是 `2026-05-12`，正确结束日期必须是 `2026-04-30`。

要求：

- 前端展示、提交 payload、worker 实际使用的时间必须一致。
- 默认时间范围：
  - 起点固定：`2023-01-01 00:00:00`。
  - 终点动态：当前月份的上个月最后一天 `23:59:59`。
  - 当前日期为 `2026-05-12` 时，必须显示并提交 `2026-04-30`。
- 不允许硬编码旧年份或构造受本地浏览器时区影响导致少一年。
- 页面要展示自动切分：训练 / 验证 / 测试，不要求用户手动输入日期。
- 高级模式可以保留手动覆盖，但默认必须隐藏或弱化，不应让用户觉得必须填写。

### 2.2 参数输入过多

用户目标不是手动猜参数，而是自动搜索。

默认模式只允许/只要求填写：

- 币种白名单，最多 20 个。
- 可选黑名单。
- 市场：现货 / U 本位合约。
- 方向：只做多 / 只做空 / 多空同时。
- 风险档位或预设参数：保守 / 均衡 / 激进。

默认模式不应要求用户手动填写：

- 加仓间距。
- 首单金额。
- 倍率。
- 最大层数。
- 止盈百分比。
- 移动止盈回撤。
- 杠杆。
- 单一止损数值。

这些参数必须由系统根据风险档位、币种波动、市场方向生成搜索空间并自动回测。

高级模式可以允许用户微调搜索范围，但必须明确标注“高级参数搜索范围”，不是单一实盘参数。

### 2.3 结果不可见

用户启动回测后必须能看到：

- 任务是否已创建成功。
- 当前状态：排队中 / 运行中 / 已完成 / 失败。
- 运行进度：至少展示阶段、已处理候选数、已完成币种数、耗时。
- 失败原因：人类可读错误，而不是原始 JSON 或空白。
- 成功后结果自动刷新，不要求用户手动找任务。

完成后必须展示：

- 每个币种独立 Top 5 表格。
- 每个候选的核心参数：间距、首单金额、倍率、最大层数、止盈、移动止盈、方向、推荐杠杆。
- 每个候选的核心指标：总收益、最大回撤、胜率或交易次数、收益回撤比、稳定性评分、过拟合标记。
- 图表：至少包含资金曲线 / 回撤曲线 / 候选对比图。没有真实数据时不能显示伪图；必须显示明确空状态或错误原因。
- 风险摘要：用中文解释该候选为什么推荐、主要风险是什么。

### 2.4 Top 5 选择与组合

结果页必须支持：

- 按币种分组显示 Top 5，而不是只显示全局 Top N。
- 每个候选可点击“加入组合”。
- 同一币种允许选择多个参数组，例如 BTCUSDT 同时选择 0.5% 间距策略和 1% 间距策略。
- 组合篮子显示所有已选策略实例。
- 每个实例可编辑：权重、杠杆、启用/停用。
- 权重合计必须实时显示，等于 100% 才允许进入发布/导出下一步。
- 推荐权重与推荐杠杆来自回测结果，但用户可以修改。
- 统计必须按“策略实例”而不是只按币种聚合，避免同币种多策略互相覆盖。

### 2.5 回测任务清理

用户已确认“清理所有任务即可”。优化实施前或实施过程中必须清理当前所有回测任务、候选、artifact 引用和任务列表数据，让回测页面从干净状态重新开始。

要求：

- 清理范围仅限马丁/回测系统数据，不触碰策略、交易、用户、账单、交易所凭证等非回测业务数据。
- 清理前先列出将清理的回测任务数量、候选数量、相关表名或存储位置。
- 如存在外部 artifact 文件，只清理本项目回测 artifact 目录，不触碰外部只读行情库 `/home/bumblebee/Project/discord_c2im/pipeline/data/market_data.db`。
- 清理后验证任务列表为空，候选列表为空。

## 3. 新的页面流程

### 3.1 第一步：创建自动搜索任务

页面标题建议：`马丁自动回测`。

默认表单只显示：

1. 币种选择：白名单最多 20 个，支持批量输入，如 `BTCUSDT, ETHUSDT`。
2. 市场选择：现货 / U 本位合约。
3. 方向选择：只做多 / 只做空 / 多空同时。
4. 风险档位：保守 / 均衡 / 激进。
5. 自动时间范围展示：`2023-01-01 → 2026-04-30`。
6. 提交按钮：`开始自动搜索 Top 5`。

高级折叠区：

- 参数搜索空间预览。
- 自定义搜索空间。
- 手动时间覆盖。
- 风险规则高级配置。

### 3.2 第二步：任务进度

创建后页面必须留在当前任务上下文，显示：

- 任务 ID。
- 状态徽章。
- 当前阶段。
- 进度条。
- 已完成币种 / 总币种。
- 已评估候选数。
- 最近更新时间。
- 错误详情或日志摘要。

### 3.3 第三步：结果分析

完成后自动展示：

- 总览卡片：参与币种、每币种 Top N、最佳组合建议、风险档位、时间范围。
- 分组结果：每个币种一个区域，里面有 Top 5。
- 候选详情抽屉或详情卡：展示参数、指标、图表、风险说明。
- 图表必须基于真实 artifact/candidate 数据。

### 3.4 第四步：组合篮子

用户点击“加入组合”后：

- 组合篮子固定在右侧或结果下方。
- 展示已选策略实例。
- 自动填入推荐权重和推荐杠杆。
- 用户可调整权重和杠杆。
- 权重总和为 100% 时显示绿色；否则显示明确提示。
- 支持移除候选。

### 3.5 第五步：组合批量应用到实盘

本轮必须完成“组合批量实盘发布 API”和对应 UI：

- 用户在组合篮子中选择多个候选，权重合计为 100% 后，点击“批量发布实盘组合”。
- 前端提交 portfolio 名称、市场、方向、风险档位、资金权重、杠杆、候选 ID 列表和每个候选的策略参数快照。
- 后端一次性创建一个 martingale portfolio，并为篮子中的每个候选生成独立策略实例或发布意图。
- 同一币种多个参数组必须生成多个独立策略实例，统计维度使用 `portfolio_id + strategy_instance_id`，不能互相覆盖。
- API 必须返回 portfolio_id、实例列表、每个实例状态、失败原因。
- UI 必须展示发布成功/部分失败/失败结果，并提供跳转到组合详情页的入口。
- 不允许按钮存在但点击后无反馈或 404。

### 3.6 第六步：实盘组合管理

发布完成后，用户必须能进入组合详情页查看：

- portfolio 基本信息：名称、市场、方向、风险档位、总权重。
- 策略实例列表：symbol、参数摘要、权重、杠杆、状态、来源候选 ID。
- 生命周期操作入口：启动、暂停、停止、删除或归档；若某些操作依赖现有策略系统能力不足，必须给出明确禁用原因。
- 统计入口：按组合、按币种、按策略实例三个层级展示，避免同币种多策略混淆。

## 4. 后端 / Worker 要求

### 4.1 任务创建接口

接口必须接收默认模式 payload：

- `symbols` / whitelist。
- `blacklist`。
- `market`。
- `direction`。
- `risk_profile`。
- `time_range_mode: auto_previous_month_end`。
- `per_symbol_top_n: 5`。

后端必须补全：

- `start_ms` = `2023-01-01 00:00:00`。
- `end_ms` = 上个月最后一天 `23:59:59`。
- train / validate / test split。
- per-symbol search spaces。

### 4.2 Worker 搜索

Worker 必须：

- 对每个币种独立生成候选。
- 对每个币种保留 Top 5。
- 不允许全局 Top N 抢占导致某些币种没有结果。
- summary 必须包含 UI 所需字段：
  - symbol
  - direction
  - spacing_bps
  - first_order_quote
  - order_multiplier
  - max_legs
  - take_profit_bps
  - trailing_take_profit_bps
  - recommended_weight_pct
  - recommended_leverage
  - parameter_rank_for_symbol
  - risk_profile
  - total_return_pct
  - max_drawdown_pct
  - score
  - overfit_flag
  - risk_summary_human

### 4.3 组合批量发布 API

必须新增或补齐组合批量发布 API，建议路径：

- `POST /backtest/portfolios/publish`：从回测候选篮子创建实盘马丁组合。
- `GET /martingale-portfolios`：列表查询组合。
- `GET /martingale-portfolios/{id}`：组合详情与实例列表。

发布请求必须包含：

- portfolio name。
- task_id。
- market。
- direction。
- risk_profile。
- total_weight_pct = 100。
- items[]：candidate_id、symbol、weight_pct、leverage、enabled、parameter_snapshot。

发布校验必须包含：

- 权重合计等于 100%。
- 每个 candidate_id 属于同一个已完成任务，且存在可发布参数。
- leverage 在市场允许范围内。
- 同一 symbol 可出现多次，但每个 item 必须生成不同 strategy_instance_id。
- 任何失败必须返回人类可读错误。

持久化必须支持：

- portfolio 主记录。
- portfolio item / strategy instance 明细。
- candidate 到 strategy instance 的来源映射。
- 发布审计字段：created_by、created_at、source_task_id。

### 4.4 图表数据

每个候选必须能返回或定位：

- equity curve。
- drawdown curve。
- trade/event summary。

如果 artifact 缺失，API 必须返回人类可读错误，UI 显示“图表数据缺失：原因”。

## 5. 验证标准

完成后必须实际跑一个双币种回测，默认建议：`BTCUSDT, ETHUSDT`。

验收标准：

1. 页面显示自动时间：`2023-01-01 → 2026-04-30`。
2. 默认表单不要求手动输入马丁细节参数。
3. 点击开始后无 404。
4. 能看到任务进度。
5. 回测完成后 BTCUSDT 显示 Top 5。
6. 回测完成后 ETHUSDT 显示 Top 5。
7. 至少一个候选能看到真实图表。
8. 候选能加入组合篮子。
9. 组合篮子能编辑权重/杠杆。
10. 权重合计 100% 后可以点击“批量发布实盘组合”。
11. 批量发布 API 成功创建 portfolio 与多个策略实例；同币种多参数组不会互相覆盖。
12. 发布后能进入组合详情页查看 portfolio 和实例列表。
13. 当前所有旧回测任务与候选已清理，任务列表从空状态开始。
14. 前端 build、相关 contract tests、worker tests、API tests 通过。

## 6. 非目标

- 不承诺收益率稳定或盈利保证。
- 不承诺 GPU 加速。
- 不允许伪造回测结果或图表。
- 不承诺发布后立即自动启动真实交易；启动实盘仍可要求用户在组合详情页二次确认。
