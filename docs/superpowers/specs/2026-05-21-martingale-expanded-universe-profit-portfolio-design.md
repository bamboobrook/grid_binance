# 马丁扩展币种收益优先深搜与组合降波动设计

**状态:** 待用户确认  
**日期:** 2026-05-21  
**执行方式:** 待确认，确认 spec 后再写计划

## 背景

当前马丁回测已经修复了杠杆保证金语义、`long_short` 真双向搜索、组合硬回撤限制和 7 币种验证。最近一次 7 币种验证中，激进档 Top1 组合达到约 `43.95%` 年化、`29.32%` 最大回撤，已经接近目标，但仍有两个提升空间：

1. 单策略候选仍可能漏掉更优参数，尤其是不同币种的波动结构差异很大，统一搜索范围不一定足够。
2. 组合器仍偏向模板枚举，未充分利用“高收益高回撤策略 + 低收益低回撤策略”的错峰回撤与低相关性来降低组合波动。

本轮目标不是承诺必然跑出年化 50%+，而是提升搜索与组合方法，让系统更有机会在真实 1m 数据、手续费、滑点、杠杆本金计算正确的前提下，找到更高收益/更低回撤的组合。

## 用户目标

用户希望通过马丁策略获得正收益，并通过多币种/多策略组合降低回撤、提高收益，核心目标为：

1. 单币种 Top10 尽量找到更高年化、更低回撤、更稳定增长的候选。
2. 组合策略在硬最大回撤限制内追求更高年化收益。
3. 组合时允许引入高收益高回撤策略，但必须通过低回撤/低相关策略分散，使组合整体回撤满足限制。
4. `long_short` 双向必须继续使用 long/short 独立参数、独立止盈止损与独立杠杆，不允许退化成单向。
5. 搜索可以花更长时间，但不能为了收益牺牲回测真实性。

## 非目标

1. 不保证任何市场阶段都能找到年化 50%+ 且回撤低于 20% 的真实策略。
2. 不使用未来函数、抽样作弊、忽略手续费、忽略滑点或用短区间结果冒充全周期结果。
3. 本轮不引入 GPU kernel 重写；优先继续利用 CPU 并行深搜。
4. 不修改无关 3000 端口服务。
5. 不改变实盘发布安全边界；回测优选结果仍需用户确认后才能进入实盘发布流程。

## 设计原则

### 1. 回撤约束优先

每个风险档位仍使用硬组合最大回撤限制：

- 保守：`20%`
- 均衡：`25%`
- 激进：`30%`

单策略可以作为“组合候选池”保留高回撤正收益策略，但最终组合结果必须满足组合级最大回撤限制。前端展示时要区分：

- `单策略合格 Top10`：单策略自身满足回撤限制。
- `组合候选池`：单策略正收益，但可能单独回撤较高，仅可参与组合优化。

### 2. 收益优先但不忽略稳定性

排序目标不是单纯最高收益。候选评分应综合：

- 年化收益率。
- 最大回撤。
- Calmar 或年化/回撤比。
- 月度收益稳定性。
- 资金曲线斜率稳定性。
- 止损次数与止损磨损。
- 手续费/滑点占收益比例。
- 交易次数是否过少或过多。

负收益候选不得进入最终可选结果，也不得进入组合结果。

### 3. 完整曲线组合，而不是指标拼接

组合优化必须使用成员的完整资金曲线按权重合成，再计算组合收益、年化、最大回撤、回撤曲线和交易成本。不得只用单策略总收益/最大回撤做静态拼接。

### 4. 高低风险互补

组合器需要显式支持以下结构：

- 高收益高回撤策略承担部分收益弹性。
- 低收益低回撤策略承担稳定器角色。
- 多币种分散降低同一币种极端行情风险。
- 同币种多参数策略允许共存，但单币种总权重不得超过 `80%`。
- 优先选择回撤时点错峰或收益曲线相关性较低的成员。

## 参数搜索设计

### 单策略搜索范围

本轮扩大但仍受控地搜索以下参数：

- 杠杆：`2x` 到 `10x`。
- 加仓间隔：约 `0.35%` 到 `6%`。
- 加仓倍率：约 `1.15` 到 `2.4`。
- 最大层数：`3` 到 `9`。
- 止盈范围：约 `0.3%` 到 `3%`。
- 尾部止损/全局止损：按策略总计划保证金风险搜索，避免止损过近导致高频磨损。
- 冷却时间/再入场节奏：必要时纳入搜索，避免同一区间反复开平。
- `long_short` 权重：多空基础权重可搜索，但 long/short 参数必须独立生成。

### 分阶段搜索

采用三阶段搜索，降低漏掉正收益策略的概率：

1. **粗筛**：覆盖宽参数空间，用代表性窗口快速淘汰明显劣质参数。
2. **精筛**：对粗筛优胜参数邻域扩展，跑更完整的数据窗口。
3. **全周期确认**：对候选池跑 `2023-01-01` 到上个月末的完整 1m 回测，并输出正式指标。

粗筛不得只看参数列表前缀，必须覆盖高杠杆、宽间隔、低倍率、高止盈、不同止损区间等尾部组合，避免再次出现收益候选被漏掉。

### 过拟合控制

正式候选必须保留分段验证指标：

- 全周期收益。
- 近一年收益。
- 最近 6 个月收益。
- 月度胜率。
- 最差季度表现。
- 样本内/样本外差异。

如果候选只靠单一短区间暴利，整体稳定性差，应降低排名或标记风险。

## 组合优化设计

### 候选池

组合候选池来自每个币种的正收益候选，分为：

- 单策略合格候选：自身最大回撤满足风险限制。
- 组合专用候选：自身最大回撤超过风险限制，但年化收益高，可能通过组合降波动。

每个币种至少尽量保留：

- Top10 单策略合格候选。
- Top10 组合专用高收益候选。
- Top5 低回撤稳定候选。

如果某个币种没有正收益候选，记录诊断原因，不强行加入组合。

### 组合权重搜索

组合器需要从模板枚举升级为“风险约束组合搜索”：

1. 先按币种、方向、相关性和收益档位做候选分层。
2. 生成多成员组合，成员数可为 `2` 到 `8`。
3. 权重粒度从粗到细搜索，例如先 `5%` 粒度，再对 Top 组合做 `1%/2.5%` 邻域优化。
4. 单币种总权重硬限制 `<=80%`。
5. 组合最大回撤硬限制 `<= 风险档位限制`。
6. 排名目标优先：年化收益/最大回撤比，其次年化收益，再其次稳定性。

### 相关性与错峰回撤

组合器应计算成员之间的曲线关系：

- 日收益相关性或采样收益相关性。
- 最大回撤发生区间是否重叠。
- 同一市场暴跌/暴涨期间成员是否同时亏损。

在收益接近时，优先选择相关性更低、回撤错峰更明显的组合。

### 输出结果

组合结果至少展示 Top10，前端默认突出 Top3。每个组合必须包含：

- 年化收益率。
- 总收益率。
- 最大回撤。
- Calmar / 年化回撤比。
- 月度收益表。
- 资金曲线。
- 回撤曲线。
- 成员列表、币种、方向、杠杆、核心参数、权重。
- 每个成员对组合收益和回撤的贡献。
- 是否满足单币种 `80%` 权重限制。

## 前端展示要求

本轮如果涉及前端，应保持人类可读：

1. 增加“扩展币种深搜”说明，告知会使用更多币种和更久耗时。
2. 清晰区分单策略 Top10、组合候选池、组合 Top10。
3. 对组合展示“为什么入选”：例如高收益成员、低回撤稳定器、相关性低、回撤错峰。
4. 回测结果中显示每个成员权重与单币种总权重。
5. 如果没有达到年化 50%，要展示最接近目标的组合和未达成原因，例如回撤限制、手续费磨损、候选不足。

## 验收标准

### 数据与搜索

1. 默认扩展池至少包含 18 个完整历史币种。
2. 每个币种使用 `2023-01-01` 到上个月末的完整 1m 数据做正式确认。
3. `long_short` 候选必须同时包含 long 与 short 策略腿，且参数可不同。
4. 正式候选必须包含杠杆、计划保证金、年化收益、最大回撤、交易次数、手续费、滑点、资金曲线、回撤曲线。

### 单策略

1. 每个有正收益结果的币种尽量输出单策略 Top10。
2. 单策略 Top10 必须自身满足当前风险档位回撤限制。
3. 另输出组合候选池，用于解释为什么某些高收益高回撤策略可参与组合但不属于单策略合格 Top10。

### 组合策略

1. 输出组合 Top10，至少 Top3 详细展示。
2. 每个组合必须包含至少两个成员，不能用单策略冒充组合。
3. 单币种总权重必须 `<=80%`。
4. 组合最大回撤必须 `<=` 风险档位限制。
5. 组合指标必须由成员完整资金曲线按权重合成。
6. 排名应优先选择收益/回撤比高且增长更稳定的组合。

### 回归验证

至少执行两类验证任务：

1. 7 币种基准：`BTC ETH SOL BNB XRP DOGE ADA`，用于和当前约 `43.95%` 年化、`29.32%` 回撤结果比较。
2. 18 币种扩展池：完整历史币种池，检验是否能找到更高收益或更低回撤组合。

验证报告必须说明：

- 是否超过 7 币种基准收益。
- 是否降低最大回撤。
- 是否找到年化 `50%+` 的组合。
- 如果没有，最高年化是多少，限制因素是什么。

## 风险与缓解

1. **搜索耗时过长**：继续使用 CPU 并行，分阶段筛选，先粗筛再全周期确认。
2. **候选爆炸导致内存高**：按币种/收益档/回撤档分层裁剪，不把所有粗筛结果都放入组合器。
3. **过拟合**：保留分段验证、近一年/近半年指标、月度收益表。
4. **组合看似降回撤但实盘同步亏损**：加入相关性和回撤重叠惩罚。
5. **收益目标不现实**：系统必须真实展示最优结果和原因，不允许为了达到目标放宽回撤或隐藏亏损。
