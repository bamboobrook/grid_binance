# 马丁动态多空配比与组合优化回测设计

**状态:** 已确认，实施核对中  
**日期:** 2026-05-14  
**目标分支:** `feature/full-v1`  
**背景:** 当前回测系统已经能完成单策略 Top10 筛选，但 `long + short` 仍偏向静态候选组合，做空侧风险没有被充分建模；用户要求将动态多空配比、组合止损、组合级资金分配优化提前纳入回测，并保证回测逻辑可连贯应用到实盘。

## 1. 目标

本轮目标是把马丁回测从“单策略 Top10”升级为“动态多空策略包 + 组合级资金分配优化”：

1. 在 `long + short` 模式下，系统不再简单固定多空比例，而是根据历史行情动态判断市场状态，输出随时间变化的 `long/short` 目标权重。
2. 动态配比模型必须同时适用于回测和实盘配置，避免回测使用一套逻辑、实盘无法复现。
3. 做空侧必须使用更严格的风险约束，因为上涨理论上没有上限，而下跌最多 100%。
4. 每个币种输出单币种 Top10 动态策略包。
5. 系统再从所有币种 Top10 中进行组合优化，输出组合级 Top10。
6. 组合资金权重不能平均分配，必须根据收益、回撤、收益回撤比、稳定性、相关性、止损频率、调仓磨损综合优化。
7. 前端必须能解释动态配比过程，包括多空权重曲线、市场状态、调仓次数、强平次数、交易成本和资金曲线。

## 2. 已确认需求

### 2.1 止损与强平模型

做空侧默认启用组合止损：

- `StrategyDrawdownPct`：策略级回撤百分比止损。
- `Atr`：ATR 动态止损。
- 两者谁先触发，谁先执行。

极端趋势强平采用组合触发：

- 指标极端触发：BTC 大盘和本币同时进入强趋势，且该趋势对当前方向不利。
- 亏损阈值触发：某方向浮亏达到该方向预算阈值。
- 二者满足其一即可触发极端风险动作。

亏损阈值不按“最大回撤限制的一半”计算，而是围绕用户设置的最大回撤限制本身搜索。例如最大回撤限制为 `20%` 时，可以搜索 `12% / 16% / 20% / 24% / 28%` 这一类阈值档位。实际档位应按风险画像和最大回撤限制生成，但必须避免过早、过频繁止损。

### 2.2 动态配比模型

动态配比采用：

- 本币趋势判断。
- BTC 大盘过滤。
- 不参考资金费率。
- 4H 主判断。
- 1D 大方向过滤。
- 每 4H 评估一次。
- 调仓后进入冷却期，避免频繁变化。
- 权重候选使用粗粒度：`100/0`、`80/20`、`60/40`、`50/50`、`40/60`、`20/80`、`0/100`。

BTC 大盘过滤规则：

- BTC 强上涨：降低或关闭所有币种 short 权重。
- BTC 强下跌：降低或关闭所有币种 long 权重。
- BTC 震荡：允许 long 和 short 同时参与。

本币趋势规则：

- 本币强上涨：增加 long 权重，降低 short 权重。
- 本币强下跌：增加 short 权重，降低 long 权重。
- 本币震荡：允许多空同时运行。
- 本币高波动：降低总风险，收紧止损，减少高风险方向权重。

### 2.3 已有仓位处理

动态权重变化时，已有仓位采用分级处理：

1. 普通趋势变化：暂停高风险方向新开仓或新加仓。
2. 强趋势变化：撤销高风险方向挂单，并继续保留已有仓位等待止盈或止损。
3. 极端趋势变化：允许强制平掉高风险方向已有仓位。

回测必须模拟这些动作，并记录事件：

- `allocation_rebalance`：目标权重变化。
- `direction_paused`：暂停某方向新开仓或加仓。
- `direction_orders_cancelled`：撤销某方向挂单。
- `direction_forced_exit`：极端趋势强制平仓。

`direction_forced_exit` 必须计入手续费、滑点和实际盈亏，不能只是虚拟降低权重。

### 2.4 交易成本

当前回测已经有基础成本模型：手续费和滑点会在开仓、加仓、止盈、止损退出时扣除。本轮新增动态配比后，还必须覆盖：

- 暂停新开仓：无直接交易成本，但记录次数。
- 撤销挂单：无成交成本，但记录次数。
- 极端趋势强平：扣除退出手续费、滑点和实际盈亏。
- 频繁调仓：不一定产生直接成交成本，但必须进入评分惩罚。

结果必须展示：

- 总手续费估算。
- 总滑点估算。
- 强平损耗。
- 止损损耗。
- 调仓次数。
- 平均权重持有时长。

### 2.5 最大回撤限制默认值

前端最大回撤限制根据风险档位自动显示：

- 保守：`20%`
- 平衡：`25%`
- 激进：`30%`

用户选择手动模式时，可以自行填写最大回撤限制。手动填写后，风险档位不应覆盖用户输入值，除非用户主动恢复默认。

最大回撤限制是硬约束：组合推荐结果的最大回撤必须小于或等于该值。若没有满足硬约束的候选，系统必须明确提示“未找到满足最大回撤限制的候选”，并展示最接近候选，但不能标记为推荐实盘。

## 3. 市场状态识别

### 3.1 指标集合

第一版采用透明规则，不使用黑盒模型：

- EMA 快慢线或 EMA 斜率：判断趋势方向。
- ADX：判断趋势强度。
- ATR：判断波动风险。
- BTC 4H / 1D 趋势：作为全局方向过滤。
- 本币 4H / 1D 趋势：作为局部方向判断。

资金费率不纳入模型。

### 3.2 状态分类

每个评估周期输出市场状态：

- `strong_uptrend`
- `uptrend`
- `range`
- `downtrend`
- `strong_downtrend`
- `high_volatility`
- `extreme_risk`

状态应同时包含：

- BTC 全局状态。
- 本币局部状态。
- 最终方向建议。
- 推荐权重档位。
- 是否允许绕过冷却期。
- 是否触发暂停、撤单或强平。

### 3.3 冷却与切换门槛

每 4H 评估一次市场状态，但权重不应频繁变化：

- 保守默认冷却：24H。
- 平衡默认冷却：16H。
- 激进默认冷却：12H。
- 冷却期内不因轻微信号变化调仓。
- 极端风险可以绕过冷却。
- 权重变化必须满足状态变化门槛，例如从震荡变为强上涨，或 BTC 与本币同向确认。

## 4. 单币种动态策略包搜索

### 4.1 搜索对象

每个币种必须独立搜索：

- Long 参数空间。
- Short 参数空间。
- Long/Short 动态权重表。
- 方向止损阈值。
- ATR 止损倍数。
- 策略回撤止损阈值。
- 调仓冷却时间。
- 极端趋势强平阈值。

Short 侧默认更严格：

- 更低杠杆。
- 更少最大层数。
- 更宽加仓间距。
- 更严格止损。
- 更高趋势确认门槛。

Long 侧可以相对宽松，但仍必须受最大回撤限制约束。

### 4.2 单币种 Top10

每个币种输出 Top10 动态策略包，每个策略包至少包含：

- symbol。
- market。
- risk_profile。
- direction_mode。
- long 参数摘要。
- short 参数摘要。
- 动态权重规则摘要。
- 推荐杠杆。
- 最大回撤。
- 总收益。
- 年化收益率。
- 收益回撤比。
- 交易次数。
- 止损次数。
- 强平次数。
- 调仓次数。
- 平均权重持有时间。
- 手续费与滑点估算。
- 资金曲线。
- 回撤曲线。
- 多空权重曲线。
- 市场状态时间线。
- 是否满足最大回撤限制。
- 是否可推荐实盘。

排序规则：

1. 先过滤不满足最大回撤限制的候选。
2. 在满足约束的候选中，优先按收益回撤比排序。
3. 收益回撤比接近时，选择总收益更高者。
4. 若调仓次数、强平次数或止损频率过高，降权。
5. 若跨 train / validate / test 表现差异过大，降权并标记过拟合风险。

## 5. 组合级优化

### 5.1 组合输入

组合优化从所有币种 Top10 动态策略包中选取候选。用户选择 10 个币种时，系统不应强制全部分配资金：

- 回测差的币种可以被舍弃。
- 回测好的币种可以分配更高资金。
- 同一个币种可以入选多个策略包，但必须受限制。

### 5.2 同币种多策略限制

同一币种允许多个策略包，但必须有限制：

- 默认单币种最多 3 个策略包。
- 单币种总资金上限：
  - 保守：25%。
  - 平衡：35%。
  - 激进：50%。
- 单策略包资金上限：
  - 保守：10%。
  - 平衡：15%。
  - 激进：25%。
- 同币种多个策略包如果收益曲线高度相似，必须降低组合权重或只保留其中一个。

### 5.3 权重搜索

组合资金权重采用：

1. 先以 10% 粒度粗搜。
2. 再围绕高质量组合以 5% 粒度细搜。
3. 权重合计必须为 100%。
4. 不要求所有币种入选。
5. 不允许单个币种或单个策略包突破风险档位上限。

### 5.4 组合 Top10

输出组合级 Top10，每个组合至少包含：

- portfolio_candidate_id。
- 入选币种列表。
- 入选策略包列表。
- 每个策略包资金权重。
- 每个策略包推荐杠杆。
- 组合总收益。
- 组合年化收益率。
- 组合最大回撤。
- 组合收益回撤比。
- 组合资金曲线。
- 组合回撤曲线。
- 组合多空总体权重曲线。
- 各币种资金占比图。
- 各策略包收益贡献。
- 各策略包回撤贡献。
- 组合调仓次数。
- 组合强平次数。
- 总手续费与滑点估算。
- 过拟合风险标记。
- 是否满足最大回撤限制。
- 是否可推荐实盘。

组合排序规则：

1. 最大回撤限制是硬约束。
2. 满足硬约束后，优先收益回撤比。
3. 收益回撤比接近时，优先总收益。
4. 对高相关、频繁调仓、频繁强平、止损磨损过高的组合扣分。
5. 对 train / validate / test 稳定的组合加权。

## 6. 前端体验

### 6.1 创建任务

默认表单保留用户易懂字段：

- 白名单币种。
- 黑名单币种。
- 市场。
- 方向。
- 风险档位。
- 最大回撤限制。
- 输出数量：每币种 Top10 + 组合 Top10。

默认不要求用户手动填写：

- Long/Short 资金比例。
- EMA / ADX / ATR 参数。
- 止损阈值。
- 强平阈值。
- 调仓冷却时间。

高级模式可以展示并允许修改这些搜索范围，但必须明确标注为“搜索范围”，不是单一实盘参数。

### 6.2 结果页

结果页必须同时展示：

- 每币种 Top10 动态策略包。
- 组合级 Top10。
- 当前选中的策略包详情。
- 当前选中的组合详情。

新增图表：

- 资金曲线。
- 回撤曲线。
- 多空权重曲线。
- 市场状态时间线。
- 组合资金分配饼图或条形图。
- 交易成本分解图。

多空权重曲线必须支持鼠标悬浮查看：

- 时间。
- Long 权重。

- BTC 状态。
- 本币状态。
- 触发原因。
- 是否处于冷却期。

### 6.3 人类可读解释

每个推荐结果必须给出中文解释：

- 为什么推荐这个策略包或组合。
- 哪些币种被舍弃，原因是什么。

- 为什么某个币种入选多个策略包。
- 做空侧触发过哪些风控。
- 调仓是否频繁。
- 交易成本是否过高。
- 是否建议直接发布实盘。

## 7. 实盘连贯性

### 7.1 发布内容

如果用户从动态组合 Top10 发布实盘，发布的不应只是固定权重，而是完整动态策略包：

- 单策略参数。
- 动态权重规则。
- 市场状态识别规则。
- BTC 大盘过滤规则。
- 调仓冷却设置。
- 止损规则。
- 极端趋势强平规则。
- 组合资金权重上限。
- 分级处理规则。

### 7.2 实盘执行第一版

实盘执行应与回测逻辑保持一致，但默认采用安全渐进方式：

- 支持自动暂停高风险方向新开仓或加仓。
- 支持撤销高风险方向挂单。
- 支持极端趋势下按规则强制平仓。
- 所有自动动作必须记录审计日志。
- 前端必须展示最近一次动态配比决策和执行动作。

如果某些执行能力依赖现有交易引擎尚未具备，发布前必须明确标记“不满足实盘动态执行能力”，不能让用户误以为回测策略已经能完整实盘复现。

### 7.3 风险保护

实盘动态执行必须有保护：

- 最大单次强平金额限制。
- 每日最大强平次数限制。
- 每日最大调仓次数限制。
- API 失败时不重复盲目执行。
- 交易所返回异常时进入安全暂停状态。
- 所有动态调仓动作必须可追踪、可解释、可回滚到暂停模式。

## 8. 数据与计算要求

### 8.1 行情数据

继续使用只读行情库：

`/home/bumblebee/Project/discord_c2im/pipeline/data/market_data.db`

要求：

- 可以读取，不允许修改。
- 必须确认 BTCUSDT 4H / 1D 数据可用于大盘过滤。
- 若原始数据粒度不是 4H / 1D，系统应从已有 K 线聚合，不应要求用户手动准备。
- 如果某币种数据不足，必须在结果中明确标记并降低或排除候选。

### 8.2 计算策略

用户选择质量优先：

- 可以牺牲部分运行时间换取更完整搜索。
- 必须有进度展示，避免用户误以为卡死。
- 任务应分阶段：单币种搜索、动态权重搜索、组合粗搜、组合细搜、结果持久化。
- 每个阶段都应记录耗时和候选数量。

### 8.3 防过拟合

必须继续使用自动时间范围：

- 起点：`2023-01-01`。
- 终点：当前日期所在月份的上个月月底。

在全区间内切分：

- train。
- validate。
- test。
- stress windows。

动态配比和组合优化不能只在 train 表现好；validate / test 表现明显衰减时必须降权或标记风险。

## 9. 验证要求

后续实施必须新增或更新测试，证明：

1. 风险档位自动填充最大回撤限制：保守 20%、平衡 25%、激进 30%，手动模式不被覆盖。
2. `long + short` 动态回测会生成多空权重曲线，而不是固定单方向结果。
3. Short 候选默认包含策略回撤止损和 ATR 止损。
4. 极端趋势强平会产生事件，并计入手续费、滑点和 PnL。
5. 调仓冷却能减少频繁权重切换。
6. 单币种输出 Top10，包含动态权重、止损、强平、成本指标。
7. 组合级输出 Top10，权重合计为 100%，并遵守单币种/单策略包上限。
8. 回测差的币种可以被组合优化舍弃。
9. 同币种最多 3 个策略包，且相关性过高时被惩罚。
10. 组合排序以最大回撤硬约束为前提，再按收益回撤比和收益排序。
11. 前端图表能展示多空权重曲线，并支持 hover 查看时间、权重和状态。
12. 交易成本在开仓、加仓、止盈、止损、强平中均被计入。
13. 实盘发布 payload 包含动态配比规则，不只是固定权重。

## 10. 非目标

本轮不做以下事项：

- 不使用资金费率作为信号。
- 不引入黑盒机器学习模型。
- 不承诺 GPU 加速。
- 不修改外部行情数据库。
- 不保证任何策略“稳赚”或无风险。
- 不跳过用户确认直接实施。

## 11. 用户确认点

本规格基于以下已确认选择：

- 做空止损：组合止损，策略回撤百分比 + ATR。
- 多空资金配比：系统根据风险和回测动态调整。
- 目标函数：最大回撤限制为硬约束，满足后优先收益回撤比。
- 动态配比：回测与实盘保持同一套模型。
- 已有仓位处理：分级处理。
- 市场状态：本币趋势 + BTC 大盘过滤，不参考资金费率。
- 时间级别：4H 主判断 + 1D 过滤。
- 调仓频率：每 4H 评估 + 冷却时间。
- 权重粒度：粗粒度档位。
- 极端趋势强平：指标极端或亏损阈值满足其一。
- 强平阈值：围绕最大回撤限制进行搜索，避免频繁止损。
- 输出：每币种 Top10 + 组合级 Top10。
