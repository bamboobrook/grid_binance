# 马丁组合回测与发布指南

## 什么是马丁网格

马丁网格是一类按价格间隔分层加仓的策略。例如首单使用 10 USDT，价格每逆向移动 1% 后按 20、40、80 USDT 继续补仓，并用整体持仓均价计算止盈。平台中的 `martingale_grid` 支持现货与 USDT-M 合约、Long/Short/Long+Short、固定间隔、倍率间隔、ATR 间隔、自定义金额序列，以及多个 symbol 组合成一个 Portfolio。合约回测中，首单金额按保证金口径理解：首单保证金 10U、2x 杠杆会开出约 20U 名义仓位，收益率、回撤和资金占用按投入保证金计算，手续费和滑点仍按名义成交额扣减。

## 为什么它风险很高

马丁策略不会保证盈利。它用更大的后续仓位换取更低的整体均价，因此在长期单边行情、流动性变差、滑点扩大、资金费率不利、交易所限制或网络中断时，亏损可能快速放大。合约模式还会叠加强平风险。实盘前必须设置全局预算、单币种预算、单方向预算、单策略预算、止损和熔断，并确认逐仓/全仓、杠杆、对冲模式（Hedge Mode）等交易所设置。

## 两阶段回测如何工作

1. **K 线海选**：先用 OHLCV 数据保守模拟大量参数组合，快速淘汰超预算、止损过多、回撤过大或数据质量不足的候选。
2. **成交级精测**：对 Top N 候选使用 aggTrades/成交价格做更细的顺序回放，减少 K 线内价格顺序不确定带来的误判。

外部行情 SQLite 数据库只会以只读方式打开；系统不会在源库上建索引、迁移、VACUUM 或 checkpoint。

## 生存优先评分

评分不是简单按收益排序。系统先淘汰以下候选：爆仓/强平风险、全局回撤超限、单策略回撤超限、资金占用超预算、止损次数过高、数据完整度不足。只有通过生存筛选的候选才按收益、Calmar、Sortino、回撤、止损频率、资金利用率和交易稳定性加权排序。

## Portfolio 发布确认

回测完成后，系统会生成 Portfolio 候选。用户需要查看风险摘要、交易对（symbol）与方向、逐仓/全仓保证金模式（margin mode）、杠杆（leverage）、预算、止损、对冲模式（Hedge Mode）要求和同交易对兼容性后，再手动确认启动。系统不会自动把候选直接投入实盘。合约 Long+Short 需要对冲模式（Hedge Mode）；同一个合约交易对（futures symbol）的多个策略必须使用相同保证金模式（margin mode）与杠杆（leverage），否则发布或启动会被拒绝。

## 为什么实盘可能不同于回测

回测无法完全复现真实撮合。实盘差异可能来自：成交深度、订单排队、滑点、手续费等级、BNB 抵扣、资金费率、API 延迟、网络失败、交易所风控、最小下单额变化、暂停/恢复时的未归属订单，以及行情库缺失。上线后请先小资金验证，并持续观察 需要处理（needs-attention）/孤儿订单（orphan order）告警、回撤和资金占用。
